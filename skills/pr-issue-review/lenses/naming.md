# Naming Lens

Challenge names that make the code harder to understand or maintain.

Check:

- Function, variable, type, file, component, test, and flag names
- Whether names reveal intent, domain meaning, units, cardinality, and lifecycle
- Whether names are too vague, too broad, misleading, overloaded, or inconsistent with existing terms
- Whether boolean names read clearly at call sites
- Whether new names match established repo vocabulary

Prefer concrete replacement suggestions when a better name is obvious.
